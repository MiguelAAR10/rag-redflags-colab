# Spec 001 — Product Brief

## Problema

Construir un RAG agentico que pueda responder preguntas complejas sobre documentos, bases de conocimiento o proyectos, usando evidencia verificable y memoria externa.

## Usuario objetivo

Definir:
- equipo interno
- cliente
- área de negocio
- tipo de documentos
- frecuencia de uso

## Caso de uso inicial

Completar:
- Qué pregunta quiere resolver el usuario.
- Qué documentos usa.
- Qué output espera.
- Qué decisión toma con ese output.

## Criterios de aceptación

- La respuesta cita fuentes.
- La respuesta declara incertidumbre.
- El sistema rechaza responder si no hay evidencia.
- Se registra la ejecución.
- Se puede reproducir el retrieval.
